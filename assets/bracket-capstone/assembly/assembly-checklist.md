# Assembly Checklist

本清单用于系统验证 V8A 装配体（`bracket_assembly.py`）的完整性。

> **使用说明**：在提交作品集前逐项检查。带 ✅ 的为必填项。

## 基本信息

- [ ] 装配体名称：
- [ ] CadQuery 代码路径：
- [ ] 提交日期：
- [ ] 维护者：

## 组件完整性

- [ ] ✅ **能否识别所有组件** — 打开 STEP，目视确认每个组件可见
- [ ] ✅ **组件数量是否正确** — 对比 BOM 表格，确认数量一致
- [ ] ✅ **螺栓/销钉位置是否合理** — 螺栓应穿过底板孔，销钉应穿过立板孔
- [ ] ✅ **组件之间是否有明显干涉** — 目视检查是否有重叠 / 穿模
- [ ] ✅ **坐标系是否清楚** — 所有组件的 Location 都以装配体原点为参考
- [ ] ✅ **能否解释每个组件的作用** — 团队 / 评审者能理解每个零件的功能

## 命名规范

- [ ] ✅ 组件命名语义化（`base_plate` 而非 `part1`）
- [ ] ✅ 命名与 BOM 表格一致
- [ ] ✅ 命名与 Assembly 代码中 `name=` 参数一致
- [ ] 颜色区分合理（不同组件用不同颜色便于识别）

## 加工分离

- [ ] ✅ **知道哪些零件需要单独加工** — 底板 / 立板加工，螺栓 / 销钉外购
- [ ] ✅ **区分展示装配体和加工模型** — 装配体用于展示，加工要回到单件
- [ ] 单件 STEP 文件已生成（如有需要）

## 导出检查

- [ ] ✅ **用 FreeCAD 打开 STEP，目视验证** — 确认所有组件位置正确
- [ ] ✅ STEP 文件无损坏（能正常打开）
- [ ] ✅ STEP 单位正确（毫米 SI_UNIT）
- [ ] ✅ STL 三角面片数量合理（无明显异常）
- [ ] 没有警告或错误信息

## 文档同步

- [ ] ✅ BOM 表格已更新
- [ ] ✅ 装配说明已记录（`assembly-notes-template.md`）
- [ ] ✅ 设计修改记录已保存
- [ ] ✅ 导出记录已保存

## 评审

- [ ] ✅ 提交者完成所有 ✅ 项
- [ ] ✅ 评审者（爸爸 / 团队）目视检查通过
- [ ] ✅ 评审者阅读 BOM 表格，理解装配关系
- [ ] 评审者提出修改建议（如有）

## 常见问题

| 问题 | 解决方法 |
|------|----------|
| 组件位置错位 | 检查 `Location(Vector(...))` 计算 |
| 组件数量不对 | 检查 `assembly.add()` 调用次数 |
| 螺栓穿模 | 检查 `bolt_length` 与 `base_thickness` |
| 销钉位置错 | 检查 `pin_length` 与 `vertical_thickness` |
| 干涉严重 | 用 FreeCAD 测量关键距离 |
